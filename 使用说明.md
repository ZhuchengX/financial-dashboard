# 财务系统数据导入导出使用说明

## 一、功能概述

财务系统已集成数据导入导出功能，支持以下操作：

1. **导出全部数据** - 将系统中的所有财务数据导出为 JSON 文件
2. **导入全部数据** - 从 JSON 文件导入完整的财务数据
3. **导入课程配置** - 仅导入课程配置（PresetData）
4. **导入现金记录** - 仅导入现金收支记录（Expenses）

---

## 二、界面位置

数据导入导出功能位于财务系统的最上方，显示为一个独立的卡片组件，包含以下按钮：

- 📥 **导出全部数据** - 蓝色按钮
- 📤 **导入全部数据** - 绿色按钮
- 📚 **导入课程配置** - 紫色按钮
- 💰 **导入现金记录** - 橙色按钮

---

## 三、使用步骤

### 3.1 导出数据

**步骤**:
1. 点击"导出全部数据"按钮
2. 浏览器会自动下载一个 JSON 文件，文件名格式为 `finance_data_YYYY-MM-DD.json`
3. 将文件保存到安全位置作为备份

**用途**:
- 定期备份财务数据
- 在不同设备间转移数据
- 与他人共享数据

### 3.2 导入全部数据

**步骤**:
1. 选择导入模式（见下方说明）
2. 点击"导入全部数据"按钮
3. 选择要导入的 JSON 文件
4. 等待导入完成，系统会显示导入统计信息

**导入模式**:
- **覆盖现有数据**: 导入的数据将完全替换系统中的现有数据
- **合并数据**: 导入的数据将添加到现有数据中（不删除现有数据）

**注意**: 导入前建议先导出现有数据作为备份！

### 3.3 导入课程配置

**步骤**:
1. 点击"导入课程配置"按钮
2. 选择包含课程配置的 JSON 文件
3. 系统会更新课程配置，其他数据保持不变

**用途**:
- 更新课程库
- 添加新的课程配置
- 同步不同团队的课程设置

### 3.4 导入现金记录

**步骤**:
1. 选择导入模式（覆盖或合并）
2. 点击"导入现金记录"按钮
3. 选择包含现金记录的 JSON 文件
4. 系统会更新现金记录，其他数据保持不变

**用途**:
- 导入历史现金流数据
- 整合来自 Excel 的现金记录
- 批量更新费用记录

## 四、数据转换

### 4.1 从 Excel 转换数据

如果你需要从 Excel 文件（如《课程配置（汇总）.xlsx》和《现金收支表(公帐).xlsx》）转换数据，请使用转换脚本：

**脚本位置**: `scripts/convert_excel_to_json.py`

**使用方法**:
```bash
# 确保 Excel 文件在项目根目录中
python3 scripts/convert_excel_to_json.py
```

**输出**: 
- `converted_data.json` - 转换后的数据文件

**转换内容**:
- 课程配置（从《课程配置（汇总）.xlsx》）
- 现金记录（从《现金收支表(公帐).xlsx》的"现金日记帐" Sheet）
- 自动学生匹配（从摘要中提取人名）

### 4.2 转换脚本功能

转换脚本会自动：
1. 读取 Excel 文件
2. 验证数据格式
3. 提取人名并匹配学生
4. 生成标准 JSON 格式
5. 保存为 `converted_data.json`

**转换统计**:
- 课程配置: 228 门课程
- 现金记录: 121 条记录
- 学生匹配: 15 条记录

## 五、常见操作流程

### 5.1 首次导入数据

1. 准备 Excel 文件：
   - `课程配置（汇总）.xlsx`
   - `现金收支表(公帐).xlsx`

2. 运行转换脚本：
   ```bash
   python3 scripts/convert_excel_to_json.py
   ```

3. 导入转换后的数据：
   - 点击"导入全部数据"
   - 选择 `converted_data.json`
   - 选择"覆盖现有数据"模式
   - 点击导入

4. 验证数据：
   - 检查课程配置是否正确
   - 检查现金记录是否完整
   - 验证学生匹配是否准确

### 5.2 定期备份

1. 每周导出一次数据：
   ```
   点击"导出全部数据"按钮
   ```

2. 将文件保存到备份位置：
   ```
   /backup/finance_data_2026-01-01.json
   /backup/finance_data_2026-01-08.json
   ...
   ```

### 5.3 更新课程配置

1. 修改 Excel 文件中的课程配置

2. 运行转换脚本：
   ```bash
   python3 scripts/convert_excel_to_json.py
   ```

3. 导入课程配置：
   - 点击"导入课程配置"
   - 选择 `converted_data.json`
   - 系统会更新课程库

### 5.4 合并多个数据源

1. 导出现有数据作为备份

2. 导入第一个数据源：
   - 选择"覆盖现有数据"模式
   - 导入第一个 JSON 文件

3. 导入第二个数据源：
   - 选择"合并数据"模式
   - 导入第二个 JSON 文件

4. 验证合并结果

## 六、数据格式说明

### 6.1 导出的 JSON 结构

```json
{
  "students": [...],           // 学生列表
  "teacherRecords": [...],     // 教师成本记录
  "expenses": [...],           // 现金记录
  "bonusRules": [...],         // 分红规则
  "teachers": [...],           // 教师库
  "initialBalance": 0,         // 初始资金
  "presetData": {...},         // 课程配置
  "exportedAt": "ISO时间戳"    // 导出时间
}
```

### 6.2 课程配置结构

```json
{
  "presetData": {
    "专业名称": {
      "年级": {
        "学期": [
          {
            "name": "课程名称",
            "type": "课程类型",
            "hours": 总课时,
            "rate": 单价,
            "defaultTeacher": "默认老师",
            "defaultCost": 默认课酬
          }
        ]
      }
    }
  }
}
```

### 6.3 现金记录结构

```json
{
  "expenses": [
    {
      "id": "exp_xxxxx",
      "date": "YYYY-MM-DD",
      "payee": "收款人",
      "reason": "摘要",
      "amount": 金额,
      "category": "分类",
      "account": "账户",
      "status": "completed",
      "projectTag": "项目标签",
      "linkStudentId": "学生ID或null",
      "handler": "经手人"
    }
  ]
}
```

## 七、故障排除

### Q1: 导入失败，提示"无效的 JSON 格式"

**原因**: JSON 文件格式不正确或已损坏

**解决方案**:
1. 检查文件是否是有效的 JSON 格式
2. 使用 JSONLint 工具验证文件
3. 确保文件编码为 UTF-8
4. 重新运行转换脚本生成新的 JSON 文件

### Q2: 导入后课程配置没有显示

**原因**: 课程配置数据可能为空或加载失败

**解决方案**:
1. 检查转换脚本是否正确运行
2. 验证 Excel 文件中是否有课程数据
3. 刷新浏览器页面
4. 查看浏览器控制台是否有错误信息

### Q3: 学生匹配不准确

**原因**: 现金记录中的人名与系统中的学生名字不一致

**解决方案**:
1. 修改 Excel 文件中的摘要，确保人名准确
2. 或在导入后手动修改现金记录的学生关联
3. 检查学生名字的拼写是否正确

### Q4: 导入后数据重复

**原因**: 选择了"合并"模式，而之前已导入过相同数据

**解决方案**:
1. 导出现有数据作为备份
2. 选择"覆盖现有数据"模式重新导入
3. 或手动删除重复的记录

### Q5: 转换脚本运行出错

**原因**: 可能是 Python 环境问题或 Excel 文件格式不正确

**解决方案**:
1. 确保已安装 pandas 和 openpyxl：
   ```bash
   pip3 install pandas openpyxl
   ```
2. 检查 Excel 文件是否为 .xlsx 格式
3. 确保 Excel 文件没有被其他程序打开
4. 查看脚本输出的错误信息

## 八、安全建议

### 8.1 数据备份

- 定期导出数据（建议每周一次）
- 保存多个版本的备份
- 将备份文件存储在安全位置
- 考虑使用云存储（如 Google Drive、OneDrive）

### 8.2 数据安全

- 导出的 JSON 文件包含所有财务数据，请妥善保管
- 避免通过不安全的通道共享数据文件
- 限制数据文件的访问权限
- 定期检查数据完整性

### 8.3 操作规范

- 导入前始终导出现有数据作为备份
- 在生产环境中谨慎使用"覆盖"模式
- 验证导入的数据是否正确
- 保留导入操作的记录

---

## 九、技术支持

如遇到问题，请检查以下内容：

1. **浏览器控制台**: 按 F12 打开开发者工具，查看 Console 标签中的错误信息
2. **文件格式**: 确保 JSON 文件格式正确，可使用 JSONLint 验证
3. **Excel 文件**: 确保 Excel 文件为 .xlsx 格式，且包含正确的列名
4. **Python 环境**: 确保已安装必要的 Python 包
